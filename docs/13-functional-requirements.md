# Tuition Management Platform - Functional Requirements

## Functional Requirements List

| Req ID | Component | Description | Priority |
|---|---|---|---|
| FR-01 | Authentication | System must support registration and login with roles: Admin, Tutor, Guardian. | Must Have |
| FR-02 | User Profile | Tutors must be able to edit education, subjects, experience, salary, and photo. | Must Have |
| FR-03 | Tuition Posting | Guardians must be able to create, edit, and delete tuition postings. | Must Have |
| FR-04 | Job Search | Tutors must be able to search and list open tuition jobs. | Must Have |
| FR-05 | Application | Tutors must be able to apply for jobs. Guardians must be able to Accept/Reject applications. | Must Have |
| FR-06 | Post Management | Accepting a tutor must automatically update the tuition post status to "Closed". | Must Have |
| FR-07 | Moderation | Admin must be able to view all users, delete posts, and block accounts. | Must Have |
| FR-08 | Reviews | Guardians must be able to leave ratings (1-5 stars) and reviews for tutors. | Should Have |
